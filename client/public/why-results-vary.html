Why Results May Vary

Understanding fluctuations helps you compare tests fairly.

Core Causes

  • Server Location: Further distance increases latency and reduces sustained throughput.
  • Parallelism: Fast links often need multiple streams; we adaptively scale concurrency.
  • Browser Limits: Connection caps and scheduling can throttle peak speeds.
  • Network Load: Competing devices/apps consume bandwidth during the test.
  • Wi‑Fi Quality: Interference and signal strength impact performance versus Ethernet.
  • ISP Routing: Peering paths differ across CDNs and can bias results.

Consistency Tips

  • Run 2–3 tests and take the median.
  • Close heavy downloads/uploads first.
  • Prefer 5 GHz Wi‑Fi or Ethernet.
  • Keep the device near the router.

Compare Fairly

Use the same device + browser for different speed sites. If another tool shows higher numbers, check if it is using a closer server or more streams.

Adaptive Concurrency

The test starts with fewer streams and increases them when growth plateaus, helping reach saturation without wasting excess data.
